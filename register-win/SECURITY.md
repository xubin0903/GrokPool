# 安全策略

## 报告安全漏洞

非常感谢你帮助提升 Grok Register Win 的安全性。如果你发现安全漏洞，请按以下流程报告，**不要在公开 Issue 中提交**。

### 报告方式

1. **GitHub 安全公告**（推荐）：前往仓库页面，点击 **Security** → **Report a vulnerability** 提交
2. **邮箱**：将漏洞详情发送至仓库维护者

### 报告内容

请尽量包含以下信息：

- 漏洞类型（如：凭据泄露、代理劫持、远程代码执行等）
- 受影响的版本号
- 复现步骤
- 影响范围与潜在危害
- 建议修复方向（可选）

## 响应流程

- **确认接收**：72 小时内回复确认
- **初步评估**：7 天内给出评估与严重等级
- **修复发布**：根据严重等级尽快发布修复版本

## 适用范围

本安全策略适用于本仓库公开发行的代码与脚本。

## 不在范围内

- 用户自行将 API Key / Token 提交至公开仓库
- 因代理配置不当导致的网络问题
- 仅需物理访问本地设备才能利用的问题
- 社会工程学攻击
